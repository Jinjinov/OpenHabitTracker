# OpenHabitTracker — Project Reference

## What it is

Free, open source app for taking Markdown notes, planning tasks, and tracking habits. No account required, no ads, no subscription. All data stays on the user's device. Optional self-hosting via Docker for cross-device sync.

Website: https://openhabittracker.net
PWA: https://pwa.openhabittracker.net
Source: https://github.com/Jinjinov/OpenHabitTracker

---

## Platforms

- Windows (Microsoft Store, direct download)
- Linux (Flathub, Snap Store, direct download)
- Android (Google Play, APKPure, direct APK)
- iOS / macOS (App Store)
- Web (PWA at pwa.openhabittracker.net)
- Self-hosted (Docker image, Blazor Server)

---

## Key features

- Markdown notes with blockquote and table support
- Tasks with checklist items, timer, start/complete times
- Habits with interval-based urgency tracking (no streak counter — tracks % of desired interval elapsed)
- Categories and priorities to organize everything
- Advanced search, filter, sort
- Weekly stats for notes, tasks, habits
- Import/export: JSON, YAML, TSV, Markdown
- Import from Google Keep Takeout ZIP
- 20 languages, 26 themes (dark + light)
- Full keyboard navigation (ARIA menu pattern, ARIA grid for calendar)
- Focus management (focus moves into sidebars, returns on close)
- Self-hosting: Docker image with JWT auth, any native app can log in and sync

## Architecture

```
OpenHabitTracker (Core)
  ├── Entities (DB POCOs)
  ├── Models (view/app models)
  ├── Services (business logic)
  ├── ClientState (state container / identity map)
  └── Startup.cs (DI registration)

IDataAccess
  ├── IndexedDB (browser/WASM)
  └── EntityFrameworkCore (SQLite, desktop/server)

OpenHabitTracker.Blazor (shared UI)
  └── Entry points: Wasm, Web, Maui, WinForms, Wpf, Photino
```

**Flow:** Component → Service → ClientState → ClientData (in-memory dicts) → IDataAccess → storage

## Data model

### Entity hierarchy

```
IdEntity { long Id }
  ├── ContentEntity { Id, CategoryId, Title, Color, Priority, IsDeleted, CreatedAt, UpdatedAt }
  │   ├── HabitEntity { RepeatCount, RepeatInterval, RepeatPeriod, Duration, LastTimeDoneAt }
  │   ├── NoteEntity { Content }
  │   └── TaskEntity { PlannedAt, Duration, StartedAt, CompletedAt }
  ├── CategoryEntity { UserId, Title }
  ├── ItemEntity { ParentId, Title, DoneAt }     — checklist items for tasks/habits
  ├── TimeEntity { HabitId, StartedAt, CompletedAt }  — habit completion records
  ├── SettingsEntity { UserId, 100+ properties }
  └── UserEntity { UserName, Email }
```

Each entity has a corresponding Model (e.g. HabitEntity → HabitModel) via `.ToModel()` extension.

### Key enums

- **Priority** — None, VeryLow, Low, Medium, High, VeryHigh
- **Period** — Day, Week, Month, Year (habit repeat period)
- **ContentType** — Note, Task, Habit
- **Ratio** — ElapsedToDesired, ElapsedToAverage, AverageToDesired
- **DataLocation** — Local, Remote

### HabitModel specifics

- Recurrence: RepeatCount, RepeatInterval, RepeatPeriod
- Tracking: TimesDone (List<TimeModel>), TimesDoneByDay (Dict<DateTime, List>)
- Calculated: TotalTimeSpent, AverageInterval, Ratio tracking
- Key methods: RefreshTimesDoneByDay(), GetRatio(), RepeatCountReached()
- **No streak counter** — urgency is expressed as % of elapsed vs desired interval

### SettingsModel

100+ boolean/enum properties. Key groups:
- UI: IsDarkMode, Theme, Culture, FirstDayOfWeek
- Display toggles: ShowPriority, ShowCategory, ShowStatistics, ShowCalendar, ShowItemList
- Filter/Sort config: FoldSection dict, SortBy dict, ShowPriorityDropdown dict
- Sync: RememberMe, DataLocation

| Service | Responsibilities |

## State management (ClientState)

ClientState is the single source of truth. Holds in-memory dictionaries per DataLocation.

```csharp
// Identity map — one dict per entity type
Dictionary<long, HabitModel>? Habits
Dictionary<long, NoteModel>? Notes
Dictionary<long, TaskModel>? Tasks
Dictionary<long, ItemModel>? Items     // lazy-loaded per parent
Dictionary<long, TimeModel>? Times     // lazy-loaded per habit
Dictionary<long, CategoryModel>? Categories

// Soft-deleted items
List<HabitModel>? TrashedHabits
List<NoteModel>?  TrashedNotes
List<TaskModel>?  TrashedTasks
```

**Lazy loading pattern:**
```csharp
if (Habits is null)
{
    await LoadCategories();
    await LoadTimes();
    Habits = (await DataAccess.GetHabits())
        .Select(x => x.ToModel())
        .ToDictionary(x => x.Id);
    // wire TimesDone from Times dict
}
```

Times and Items are lazy-loaded per parent (not all at once) because the Times table can grow large (180K+ records in long-running instances).

### Known bugs in state/services

1. **Orphaned Times objects** — HabitService.LoadTimesDone() loads objects not stored in the Times dict
2. **CategoryModel null sub-lists** — Notes/Tasks/Habits only wired in GetUserData() export path, not during normal load
3. **Broken DeleteCategory cascade** — does not delete related habits/tasks/notes
4. **DataAccess is public** — should be private; services currently access it directly (violates Identity Map pattern)

## Dependency injection wiring

Each entry point (e.g. `OpenHabitTracker.Blazor.Wasm/Program.cs`) calls:

```csharp
builder.Services.AddServices();       // core services (Startup.cs)
builder.Services.AddDataAccess();     // IndexedDB or EF Core
builder.Services.AddBackup();         // import/export
builder.Services.AddBlazor();         // Blazor-specific (JS interop, etc.)
// platform-specific:
builder.Services.AddScoped<IOpenFile, OpenFile>();
builder.Services.AddScoped<ISaveFile, SaveFile>();
builder.Services.AddScoped<INavBarFragment, ...>();
```

Platform abstractions (registered per entry point):
- `IOpenFile` / `ISaveFile` — file I/O
- `INavBarFragment` — platform-specific nav bar
- `IAuthFragment` — authentication UI
- `IPreRenderService` — server-side pre-rendering
- `ILinkAttributeService` — link styling
- `IAssemblyProvider` — assembly reflection

## Localization

Custom `JsonStringLocalizer` loads JSON files from `OpenHabitTracker/Localization/Resources/`.

File naming: `LANG.json` (e.g. `en.json`, `sl.json`) — one file per language shared across the whole app. Exception: `GuidedTourComponent` has its own separate files (`GuidedTourComponent-LANG.json`) because its content is too large to include in the shared files.

20 languages: English, German, Spanish, Slovenian, French, Portuguese, Italian, Japanese, Chinese, Korean, Dutch, Danish, Norwegian, Swedish, Finnish, Polish, Czech, Slovak, Croatian, Serbian

Missing keys show as `❗ KeyName ❗` in Debug, fall back to key name in Release.

**When adding localization keys:** add to ALL 20 JSON files in `OpenHabitTracker/Localization/Resources/`.

**Grammatical gender note:** "habit" in Slovenian is "navada" (feminine) — use correct gender agreement.

## Deployment

### IIS (for E2E tests and local dev)
```
dotnet publish e:/Jinjinov/OpenHabitTracker/OpenHabitTracker.Blazor.Wasm/OpenHabitTracker.Blazor.Wasm.csproj -c Release -o C:/inetpub/wwwroot
```
App runs at http://localhost. If E2E tests fail, run this command first.

### Docker (self-hosting)
```yaml
services:
  openhabittracker:
    image: jinjinov/openhabittracker:latest
    ports:
      - "5000:8080"
    environment:
      - AppSettings__UserName=${APPSETTINGS_USERNAME}
      - AppSettings__Email=${APPSETTINGS_EMAIL}
      - AppSettings__Password=${APPSETTINGS_PASSWORD}
      - AppSettings__JwtSecret=${APPSETTINGS_JWT_SECRET}
    volumes:
      - ./.OpenHabitTracker:/app/.OpenHabitTracker
```

After login at http://localhost:5000/login:
- Logs: http://localhost:5000/watchdog
- OpenAPI: http://localhost:5000/scalar/v1
